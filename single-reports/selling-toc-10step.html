🌲 Step 2｜現況實境樹（CRT）

用 If-Then 因果鏈，從10個不良效應追溯到3個根因。

不良效應清單（UDE）
#不良效應來源
UDE1向客戶提出影像專業建議，但客戶不買單簽書會現場
UDE2不確定客戶在決策旅程的哪個階段，導致說錯階段的話書 Ch5 + 簽書會
UDE3講得太專業，客戶聽不懂或覺得有「代溝」簽書會現場
UDE4自我介紹停在「我是攝影師」，沒有傳遞獨特價值簽書會 ABCDE 練習
UDE5表達中缺少「感受」成分，內容有料但沒有溫度簽書會「資訊/觀點/感受」段落
UDE6四條事業線各有各的表達方式，沒有統一的核心訊息跨對話推斷
UDE7學了很多框架（NLP、MOT、KISS…），但實戰中回歸直覺跨對話推斷
UDE8溝通前沒有做「Why-What-How」準備就開講書 Ch4
UDE9客戶覺得「不如預期」但事前沒有做期望值定錨書 Ch5
UDE10整體商業變現效率與專業能力不成正比系統最終不良效應
If-Then 因果鏈

🔴 根因 A：開口前缺乏「以終為始」的系統化思考流程

如果 溝通前沒有走 Why-What-How 序列（根因A）

那麼 → 不清楚這次對話的目標是什麼（UDE8）

那麼 → 無法判斷客戶在決策旅程的哪個階段（UDE2）

那麼 → 在客戶還沒有需求時就講產品優勢，或在需要信任時猛推方案

那麼 → 客戶覺得「講得很專業但跟我無關」（UDE3）+ 客戶不買單建議（UDE1）

那麼 → 沒有事先做期望值管理，客戶對結果失望（UDE9）

最終 → 專業能力無法轉化為商業價值（UDE10）

🔴 根因 B：多條事業線缺乏統一的核心定位與 Key Message

如果 四條事業線各自為政，沒有一個統一的「我是誰」敘事（根因B）

那麼 → 自我介紹只能停在「我是攝影師」層級（UDE4）

那麼 → 每次換場景都要重新構思表達策略，認知負荷過高

那麼 → 精力分散在多個方向（UDE6）

最終 → 沒有一條線的表達深度足以建立話語權（UDE10）

🔴 根因 C：框架多但缺乏「即時回饋→強化循環」的落地機制

如果 學了 NLP、MOT、KISS 等多套框架但沒有統一的實踐流程（根因C）

那麼 → 框架留在「知道」層面，實戰中回歸直覺式表達（UDE7）

那麼 → 表達中缺少「資訊×觀點×感受」的刻意調配（UDE5）

那麼 → 每次對話後沒有復盤，錯誤模式重複發生

最終 → 讀再多書也無法轉化為行為改變（UDE10）

根因驗證
根因消除後影響的UDE數在Rex控制範圍內？書有提供解法？槓桿度
A：缺乏以終為始思考6/10（UDE1,2,3,8,9,10）✅ 完全可控✅ Ch4 Why-What-How⭐⭐⭐⭐⭐
B：缺乏統一定位3/10（UDE4,6,10）⚠️ 需要做取捨決策⚠️ 部分（ABCDE模組）⭐⭐⭐
C：框架未落地3/10（UDE5,7,10）✅ 完全可控⚠️ 部分（Try This練習）⭐⭐⭐

結論：根因 A 是當前系統的主要制約。先集中全部資源打破它。

☁️ Step 3｜核心衝突雲（EC）

衝突雲 1（書的核心兩難）

A：讓專業價值被看見、被理解、被買單

B：展現專業深度
需要讓客戶知道你的能力、經驗、獨特性
C：建立信任關係
需要讓客戶感覺被理解、被尊重、有安全感
D：大量輸出專業資訊
滔滔不絕介紹背景、產品特色、技術優勢
⚡
D'：專注聆聽和提問
把時間給對方，先理解需求再回應

錯誤假設 α：「專業度必須靠大量資訊來證明」→ 錯！一句精準的 Key Message 勝過一小時的背景介紹。

錯誤假設 β：「聆聽 = 被動 = 浪費30分鐘會議時間」→ 錯！聆聽是最高效的情報蒐集。

錯誤假設 γ：「D 和 D' 是時間的零和博弈」→ 錯！用 Why-What-How 可以用 20% 的時間表達 80% 的關鍵資訊，剩下的時間用來聆聽。

衝突雲 2（Rex 的個人兩難）

A：建立個人品牌影響力

B：探索多元機會
同時發展影像/保險/社群/紫微，保持選項開放
C：深耕單一領域
在一個方向上建立不可替代的話語權
D：四線並行
每條線都投入一些精力，分散風險
⚡
D'：聚焦一線
砍掉其他線，全力押注一個方向

錯誤假設 δ：「聚焦 = 放棄其他線」→ 不一定！可以設定一條「主線」和多條「支線」，支線不需要砍掉，只需要降低表達投入的優先級。

錯誤假設 ε：「四條線沒有共通點」→ 錯！Rex 的所有線都圍繞「幫助專業人士/企業展現價值」，可以抽出一個統一的上層 Key Message。

書的突破注入：不是「聚焦一線」，而是「統一核心定位 + 在主線上練熟表達系統 + 複製到其他線」。

🌳 Step 5｜未來實境樹（FRT）

如果 Rex 在每次重要溝通前花 3 分鐘走 Why-What-How（I1）

加上 用 KISS 快速檢核 Key Message 是否到位（I2）

那麼 → 每次表達都有明確目標和核心訊息（消除 UDE8）

那麼 → 減少無效輸出，對話效率提升 50%+

如果 對話效率提升 加上 判斷客戶在決策旅程的哪個階段（I3）

那麼 → 說「這個階段該說的話」，不再一上來就講產品優勢（消除 UDE2）

那麼 → 客戶感覺「你懂我」而非「你在推銷我」（消除 UDE1、UDE3）

如果 客戶感覺被理解 加上 有做期望值管理（I6）

那麼 → 客戶的預期和交付品質對齊（消除 UDE9）

那麼 → 客訴率下降、口碑和轉介率上升

最終 → 專業能力開始穩定轉化為商業價值（消除 UDE10）

⚠️ 負面分支檢查
注入可能副作用預防措施
I1 Why-What-How過度思考導致「分析癱瘓」，在需要快速反應時卡住設定「重要溝通」和「日常對話」的區分標準。只有涉及商業目標的溝通才啟動完整序列，日常閒聊不啟動。
I3 決策旅程定位過度「分析」客戶而忽略「連結」，讓對方覺得被研究提問要自然融入對話（「你們目前在這方面有什麼嘗試？」），不要像問卷調查。
I5 三成分調配初期刻意加入「感受」會不自然，反而降低可信度從最真實的感受開始（不要編造），一次只加一句，逐步增加比例。
全部注入同時練 7 個注入 = 同時改 7 個習慣 = 一個都改不成TOC 紀律：一次只聚焦 1 個注入。從 I1 開始，練到自動化再加 I2。

🎯 Step 6｜Rex 的瓶頸深度定位

五型瓶頸逐一診斷

類型 A — 認知瓶頸（不相信）

診斷問題：書的哪個主張讓你本能想反駁？

診斷結果：✅ 不卡。 Rex 的理解力和接受度很高，不會因為「這是雞湯」就排斥。他在簽書會上對 KISS 框架、ABCDE 模組都展現了積極態度。唯一微弱的抗拒可能在「反話術」立場上——Rex 可能仍期待一些「具體句型」作為起步。但這不構成主要瓶頸。

類型 B — 技能瓶頸（不會做）

診斷問題：書假設你已經會什麼，但你其實不會？

診斷結果：⚠️ 中度。兩個具體技能缺口：①「判斷客戶在決策旅程哪個階段」需要通過提問來探測，但提問技巧需要練習。②「區分資訊/觀點/感受」是一個需要刻意練習才能自動化的能力。但這些都可以通過實戰累積，不是根本性障礙。

類型 C — 環境瓶頸（沒條件）

診斷問題：書假設你有什麼資源，但你其實沒有？

診斷結果：⚠️ 中度。書假設讀者有穩定的客戶會議頻率作為練習場景。Rex 的影像案件頻率不規律，可能一個月有密集提案，下個月很少。這意味著練習機會不穩定，習慣養成曲線更長。對策：把「社群貼文」也當作練習場景——每篇貼文都是一次 KISS 練習，頻率可控。

類型 D — 慣性瓶頸（舊習太強）🔴 主瓶頸

診斷問題：舊模式給你什麼即時滿足？新模式要多久才見效？

診斷結果：🔴 嚴重。

舊模式的即時回饋：直覺式表達（到了就開講、想到什麼講什麼）→ 即時滿足：「我講完了，任務完成」的控制感 + 不需要花精力做準備的輕鬆感 + 遇到不同場景可以快速切換的靈活感。

新模式的延遲回饋：系統化表達（Why-What-How → KISS → 復盤）→ 回饋延遲 3-7 天：客戶回覆、成交率變化、社群互動數據。而且初期因為「不熟練」，表達反而可能比直覺模式更卡頓。

這就是慣性瓶頸的本質：舊模式「60分的即時回饋」 vs 新模式「85分但要等一週才看到」。人腦天然選擇即時回饋。

加上多線經營的放大效應：你在影像線剛練了兩天的 KISS，就要切到保險線的溝通場景——大腦瞬間回到熟悉的舊模式。多線切換是慣性瓶頸的放大器。

類型 E — 恐懼瓶頸（怕後果）

診斷問題：如果你真的改變表達方式，你害怕失去什麼？

診斷結果：⚠️ 潛在。可能存在一個隱性恐懼：「如果我用系統化的方式表達，會不會失去自己的自然和真實？」小竺在簽書會上說「你現在的表達是過往人生的總和」——這意味著改變表達就是改變「你是誰」的一部分。Rex 可能在意識層面接受這個改變，但在潛意識層面有抵觸。對策：框架是「60分的保底」，不是「取代你的個性」。用框架的骨架，填入你自己的血肉。

Rex 的瓶頸結構：
🔴 主瓶頸 = 慣性（舊的直覺式表達太順手）
⚠️ 次瓶頸 = 技能（判斷決策階段 + 三成分調配）
⚠️ 環境 = 練習場景不規律 → 用社群貼文補充
⚠️ 恐懼 = 潛在的「失去自然感」→ 框架是骨架不是牢籠

🧱 Step 7｜障礙拆解（前提樹 PrT）

🎯 目標：讓「重要溝通前走 Why-What-How + KISS 檢核」成為不需要提醒的自動行為（≈ 30 天內）

🧱 障礙 1（慣性）：沒有「暫停」觸發器，到了就直接開講

來源：系統障礙 + 心理障礙

🏁 中間目標：在「重要溝通」和「開口」之間插入一個物理觸發點

行動：① 定義「重要溝通」（涉及商業目標的對話，包括客戶會議、提案、有目的的社交）② 手機日曆設定「溝通前 15 分鐘」提醒，內容只有一行：「Why？」③ 在皮夾/手機殼裡放一張口袋卡

🧱 障礙 2（資源）：檢核流程太長，實戰來不及

來源：知識障礙

🏁 中間目標：把 7 個注入壓縮成 5 個問題的「口袋卡」

行動：製作一張名片大小的卡（或手機備忘錄置頂），正面 5 問：❶ 我期待什麼結果？（Why）❷ 對方在決策旅程哪裡？❸ 我的 Key Message 一句話？❹ 資訊/觀點/感受比例？❺ 我希望對方下一步做什麼？（CTA）

🧱 障礙 3（慣性×環境）：多線切換讓新習慣無法固化

來源：系統障礙

🏁 中間目標：選定一條主線，30 天內只在這條線上練習新模式

行動：選擇「叡思影像提案」作為主練習場景（理由：頻率可預期、客單價最高、ROI 最明顯）。其他線暫時繼續用舊模式——不是放棄，是「先做好一條再複製」。

🧱 障礙 4（慣性）：新模式缺乏即時回饋，大腦選擇舊模式

來源：心理障礙

🏁 中間目標：為新模式設計「人工即時回饋」

行動：每次溝通後花 2 分鐘做「事後復盤 3 問」並記錄在備忘錄：① 我的 Why 達成了嗎？② 對方反應屬於四種的哪一種？③ 下次我調整什麼？——這個復盤本身就是即時的成就感。

🧱 障礙 5（恐懼）：「用框架會不會不自然？」的潛在抵觸

來源：心理障礙

🏁 中間目標：完成一次「安全實驗」——在低風險場景試用框架，體驗到「自然感沒有消失」

行動：先在社群貼文（非同步、可修改、低風險）上試用 KISS 檢核。成功一次後再推進到客戶溝通。

執行順序：障礙 3（選定主線）→ 障礙 2（製作口袋卡）→ 障礙 5（社群安全實驗）→ 障礙 1（設定觸發器）→ 障礙 4（建立復盤機制）→ 🎯 目標

🪜 Step 8｜轉變樹（TT）— 四週行動計畫

每個行動都包含四元素：現狀 → 行動 → 預期效果 → 驗證指標，加上成功/失敗的分支。

📅 第一週：地基 — 建立觸發與工具

行動 1：選定主練習線 + 製作口袋卡

現狀四條事業線各自為政，沒有統一的練習場景
行動① 決定以「叡思影像提案」為主練習線 ② 製作口袋卡（5 問），存入手機備忘錄置頂 + 實體卡放皮夾（15分鐘完成）
預期效果有了明確的「練習場」和隨身工具，降低每次「啟動新模式」的認知負荷
驗證指標口袋卡是否已存在手機+皮夾？主線是否確認？
✅ 成功：進入行動 2
❌ 失敗/卡住：如果無法選定一條線 → 先不選，直接拿「下一次任何客戶溝通」當練習場

行動 2：用 ABCDE 重寫自我介紹

現狀自我介紹停在「我是 Rex，攝影師，拍過總統和跨國企業」
行動用 ABCDE 模組重寫 30 秒版自介，寫完後錄音播放一次（20分鐘完成）
預期效果從「我是什麼」升級到「我能為你解決什麼」，並埋下商業邀請種子
驗證指標是否有一段30秒可講完的 ABCDE 文字？聽自己的錄音時是否覺得「比之前清楚」？
✅ 成功：進入行動 3
❌ 卡住：如果寫不出 D（關鍵問題）→ 回想最近三個客戶來找你的原因，那就是 D

行動 3：在社群貼文上做第一次 KISS 安全實驗

現狀社群貼文是想到什麼寫什麼，沒有用框架
行動本週發一篇貼文前，先用口袋卡跑一遍 5 個問題。寫完後檢查：有沒有一個明確的 Key Message？有沒有同時包含資訊+觀點+感受？
預期效果體驗到「用框架不會失去自然感」的安全感 + 貼文的互動數據可能有變化
驗證指標貼文是否發出？互動數據和往常相比有無變化（不求提升，只求觀察）？
✅ 成功：第一週目標達成。恐懼瓶頸初步鬆動。
❌ 卡住：如果覺得「太刻意」→ 只保留第一問「Why」，其他先不管。降低門檻。
📋 第一週驗證：口袋卡存在✓ + ABCDE 自介寫好✓ + 社群貼文安全實驗完成✓ → 全部完成才進入第二週。如果只完成 2/3，第二週繼續完成剩下的。
📅 第二週：實戰 — 在影像提案中落地

行動 4：提案前走完口袋卡 5 問

現狀提案前通常直接準備作品集和報價，沒有做「Why」思考
行動在下一次影像提案前 15 分鐘，拿出口袋卡，寫下 5 個問題的答案（特別是 ❷ 客戶在決策旅程哪裡 和 ❸ 我的 Key Message）
預期效果提案時有明確目標，不會「到了就開始秀作品集」，而是先了解客戶需求再對接
驗證指標提案前是否寫了口袋卡的答案？提案過程中是否有至少一次「先問再說」？
✅ 成功：進入行動 5
❌ 忘了/來不及：會議結束後立刻做復盤 3 問（行動 5），把這次當「對照組」

行動 5：事後復盤 3 問（每次溝通後 2 分鐘）

現狀溝通結束後沒有復盤，同樣的錯誤模式重複發生
行動每次重要溝通後打開備忘錄，寫三行：① 我的 Why 達成了嗎？② 對方反應 = 沒反應 / 蛤 / 所以咧 / 認同不行動？③ 下次調整什麼？
預期效果建立「即時回饋」替代物，讓大腦對新模式產生正向連結
驗證指標本週是否累積了 ≥ 2 份復盤筆記？
✅ 成功：慣性瓶頸開始鬆動。進入第三週。
❌ 本週沒有客戶溝通：用社群貼文的讀者回應做復盤（同樣的 3 個問題）
📅 第三週：進階 — 三成分調配 + 期望值管理

行動 6：刻意加入「感受」成分

現狀表達以資訊和觀點為主，很少加入「我的感受」（簽書會中確認的 UDE5）
行動在本週一次提案中，刻意加入一句真實感受：「說實話，看到你們品牌的潛力還沒被影像好好呈現，我覺得有點可惜。」觀察對方反應。
預期效果對方的反應從「專業但冷」變成「有溫度，想繼續聊」
驗證指標加入感受後，用四種反應診斷法觀察變化
✅ 成功：技能瓶頸開始鬆動。
❌ 覺得不自然：降級——先在社群貼文中加入感受成分，不在面對面場景嘗試

行動 7：在一次溝通中實踐期望值管理

現狀客戶對影片成品的期待常常和實際交付有落差
行動在下一次簽約前溝通中，主動說：「影片製作通常需要 X 週，中間會有兩次對稿確認。有些客戶期待一次定稿，但我經驗是兩次確認能確保品質。」
預期效果客戶預期被定錨在合理範圍，後續「不如預期」的可能性降低
驗證指標是否主動做了期望值定錨？客戶的反應是「哦，好的」（正向）還是沉默/皺眉（需調整）？
📅 第四週：系統化 — 從行動變成系統

行動 8：整理個人版「表達 SOP」

現狀前三週累積了復盤筆記和實戰經驗，但沒有整理成自己的流程
行動回顧所有復盤筆記，提取你自己版本的「提案前準備流程」——不是書的版本，是你用起來最順手的版本。寫成一頁 A4 貼在工作桌前。
預期效果從「照著口袋卡做」升級到「這是我自己的系統」——內化完成
驗證指標是否產出了一份「Rex 版表達 SOP」？下一次溝通不看口袋卡也能自動走流程？

行動 9：設計叡思影像消費決策旅程表

現狀每次遇到不同階段的客戶，都要重新思考「該說什麼」
行動畫一張表：5 個決策階段 × 每階段的溝通重點 × 具體的話術/行為。做一次，用很久。
預期效果未來遇到任何客戶，先判斷階段 → 對照表格 → 直接執行。決策速度提升 3 倍。
驗證指標表格是否完成？在下一次客戶接觸時是否有參照使用？
📋 最終驗證（第四週末）：
「重要溝通前自動走 Why-What-How」是否已成為不需要提醒的行為？
用數據回答：四週內的客戶回覆率/成交率/互動數據，和四週前相比是否有正向變化？

🔄 Step 10｜持續改善循環（POOGI）

TOC 五聚焦步驟

1. 識別當前瓶頸

慣性瓶頸 — 開口前沒有系統化思考流程，回歸直覺式表達。

2. 利用瓶頸

最大化利用現有的「直覺式表達能力」：直覺不是敵人，它是你 20 年累積的經驗值。用 KISS 框架引導直覺而非取代它 — 先想 Why（3秒），然後讓直覺帶著框架跑。

3. 遷就瓶頸

讓所有非瓶頸環節配合：暫時不要同時改善「聲音技巧」「肢體語言」「PPT設計」等非瓶頸環節。全部精力放在「建立溝通前思考的自動化」上。

4. 提升瓶頸

用四週計畫打破慣性：物理觸發器 → 口袋卡 → 復盤機制 → 個人SOP → 自動化。

5. 回到第一步 — 瓶頸是否已遷移？
