說到賣得動

不是理解更多，是打破你「知道卻做不到」的那個卡點

基於書本試閱 × 簽書會逐字稿 × 超級大腦讀書法｜為 Rex（叡思影像）量身打造

📍 Step 1｜系統界定

系統名稱：Rex 的「專業價值 → 商業變現」表達轉化系統
維度描述
輸入攝影/影像專業能力、保險知識、社群經營經驗、紫微斗數知識
產出客戶信任 → 付費委託 → 口碑轉介 → 個人品牌影響力
組成要素自我介紹、客戶提案、社群內容、一對一溝通、售後跟進
系統目標讓專業被看見、被理解、被買單 — 用最少的推銷達成最大的變現
作者對系統的三個假設

假設①：你的專業能力已經足夠好，問題出在「表達」而非「能力」。

假設②：你的表達場景主要是同步口語溝通（提案、演講、一對一對話）。

假設③：你願意投入時間做「開口前的思考準備」，而非只追求話術速成。

⚠️ Rex 的系統與作者假設的差異：

Rex 同時經營影像、保險、社群經濟、紫微斗數四條線。系統複雜度比書假設的「一人公司單一專業」高得多。最大的瓶頸可能不在「表達技巧」，而在「用表達來聚焦，而不是用行動來分散」。

🌲 Step 2｜現況實境樹（CRT）

從書 + 簽書會 + Rex 的具體情境，提取不良效應並追溯根因。

UDE7（最終不良效應）：專業很強但商業變現效率偏低

🔴 根因 A：開口前沒有完成「Why → What → How」的思考序列

🔴 根因 B：缺少統一的核心定位，多條事業線的表達各自為政

根因驗證：

如果消除根因 A（每次溝通前先走 Why-What-How 序列），UDE1、UDE2、UDE3、UDE5 都會顯著改善 — 影響 4/7 個不良效應。

如果消除根因 B（建立統一核心定位），UDE4、UDE6 會被解決，UDE7 也會因為聚焦而改善 — 影響 3/7 個不良效應。

主瓶頸判定：根因 A 是槓桿點最高的瓶頸。根因 B 是第二瓶頸，但在 A 被解決之前處理 B 效果有限。

☁️ Step 3｜核心衝突雲（EC）

書的核心洞見在解決什麼「兩難困境」？

A 共同目標：讓專業價值被客戶看見、理解、買單
B 需求①：展現專業
為了讓客戶買單，需要充分展示你的專業能力和產品/服務的價值
C 需求②：贏得信任
為了讓客戶買單，需要讓對方感覺被理解、被尊重、有安全感
D 行動①：大量輸出專業資訊
滔滔不絕介紹背景、特色、優勢、案例
⚡
D' 行動②：閉嘴、聆聽、提問
把時間給對方，先了解需求再回應
D ↔ D' 的衝突：

時間有限（30分鐘會議），你花在「說」的時間多了，「聽」的時間就少了。大量輸出專業會讓你看起來專業，但對方會覺得「你根本不在意我要什麼」。反過來，花太多時間聽，又怕對方覺得你「沒料」。

背後的錯誤假設：

假設 α：「專業度靠大量資訊來證明」— 錯！專業度靠精準的 Key Message 來證明。一句打到痛點的話 > 一小時的背景介紹。

假設 β：「聆聽 = 被動 = 浪費時間」— 錯！聆聽是最高效的情報蒐集，讓你後續的表達精準度提升10倍。

假設 γ：「展現專業和贏得信任是競爭關係」— 錯！用正確的順序（Why → What → How）可以兩者兼得。

書的核心突破：「展現專業」和「贏得信任」不是二選一。
先聆聽（贏得信任）→ 再用 KISS 精準表達（展現專業）
= 用最少的話達成最大的影響力。

💉 Step 4｜注入解萃取

注入 1：Why → What → How 思考序列 🟢 認知注入

打破假設：α（專業度靠大量資訊證明）

機制：開口前先想清楚「我期待什麼結果」→ 自然會篩選出最關鍵的訊息，減少無效輸出

前提條件：需要願意「慢下來」，接受準備時間是投資不是浪費

邊界條件：在完全即興的社交場合（如酒會），沒有時間走完整序列

注入 2：KISS 四維檢核 🟡 行為注入

打破假設：γ（展現專業和贏得信任是競爭關係）

機制：用 K-I-S-S 檢核確保表達同時具備「精準」（K+I）和「吸引力」（S+S），不用犧牲任何一方

前提條件：已理解注入1的 Why-What-How 序列

邊界條件：完全陌生的產業場景，KISS 的 I（訊息）需要額外調研

注入 3：消費決策旅程定位 🟡 行為注入

打破假設：β（聆聽 = 被動 = 浪費時間）

機制：先判斷客戶在5階段的哪個位置，再說「這個階段該說的話」。在需求未產生時引發痛點，不在還沒有需求時就講優勢。

前提條件：需要有提問的能力來判斷客戶位置

邊界條件：一次性短暫接觸（如活動社交），來不及做完整的階段判斷

注入 4：ABCDE 自我介紹模組 🟡 行為注入

打破假設：α（自我介紹 = 介紹背景經歷）

機制：用「我能為你解決什麼問題」取代「我做過什麼事」，同時在最後埋下種子（商業邀請）

前提條件：需要先完成自己的角色定位（注入5）

邊界條件：非商業場合（朋友聚會、家庭場景）不適用

注入 5：資訊 × 觀點 × 感受的三成分調配 🔴 結構注入

打破假設：「專業表達 = 給資訊」

機制：純資訊沒有獨特性，純觀點像靠北，純感受沒有可信度。三者混搭，根據對方狀態動態調整比例

前提條件：需要區分清楚自己在講的是資訊、觀點還是感受

邊界條件：需要大量練習才能自然切換，初期會有刻意感

🌳 Step 5｜未來實境樹（FRT）

如果 Rex 執行了所有注入，因果鏈長什麼樣？

注入1 Why-What-How→ 每次溝通有明確目標→ 減少無效輸出
注入3 決策旅程定位→ 說對階段的話→ 客戶感覺「被理解」
前兩者結合→ 注入2 KISS 精準表達→ Key Message 戳到痛點→ 客戶從「不理你」變成「想了解更多」
注入4 ABCDE→ 每次社交都埋下種子→ 自來客比例提升
⚠️ 負面分支檢查：

副作用①：過度思考導致行動遲緩 — 每次開口前都想走完 Why-What-How，可能讓你在需要快速反應的場合「卡住」。

預防：日常對話不用走完整序列，只在「重要對話」前啟動。用小竺的話說：模組確保「60分下限」，不是追求「100分完美」。

副作用②：專注練習表達可能吃掉「做專業」的時間 — Rex 已經同時做很多事了。

預防：不加新事，只改已有行為。每次已經要做的提案/溝通，多花5分鐘用 KISS 檢核，而非額外創造練習場景。

🎯 Step 6｜Rex 的瓶頸定位

同一本書，不同讀者的瓶頸完全不同。以下是針對 Rex 的個人化診斷。

注入認知技能環境慣性恐懼主瓶頸
Why-What-How ✅✅✅ 🔴✅ 慣性
KISS 四維檢核 ✅⚠️✅ ⚠️✅ 技能+慣性
消費決策旅程 ✅🔴✅ ✅✅ 技能
ABCDE 自介 ✅✅✅ 🔴✅ 慣性
資訊×觀點×感受 ⚠️🔴✅ ⚠️✅ 技能
🔴 Rex 的主瓶頸 = 慣性瓶頸（類型 D）

你不是「不知道」也不是「不會做」— 你接觸過大量的方法論，理解力很強。問題是：舊模式太順手了。直覺式表達（邊講邊想、直接給專業建議、跳過需求探詢）有即時回饋（「我講完了，任務完成」），而 Why-What-How 的回饋是延遲的（「客戶三天後才回我說要簽約」）。

加上你同時經營多條線，每條線的表達情境不同，很難建立統一的新習慣 — 因為你還沒來得及在影像線練熟 KISS，就切換到保險線了。

🔵 Rex 的次瓶頸 = 技能瓶頸（類型 B）

「區分資訊/觀點/感受」和「判斷客戶在決策旅程的哪個位置」是需要刻意練習的技能。書提供了框架，但把框架變成反射動作需要重複實戰。

🧱 Step 7｜障礙拆解（前提樹）

🎯 目標：每次重要溝通前自動啟動 Why-What-How + KISS 檢核

🧱 障礙 1：沒有觸發點 — 舊習慣是「到了就開始講」，沒有「暫停→思考」的機制
🏁 中間目標：建立物理觸發器
行動：手機設定「會議前15分鐘提醒」，提醒內容就是「Why？對方在決策旅程哪裡？我的 Key Message 是什麼？」
🧱 障礙 2：檢核流程太長 — KISS + 決策旅程 + 五元素加起來太複雜，實戰中來不及
🏁 中間目標：壓縮成一張「口袋卡」
行動：製作一張名片大小的檢核卡，正面只有5個問題：①Why？②對方在哪個階段？③我的 Key Message？④我要給資訊/觀點/感受的比例？⑤我希望對方下一步做什麼？
🧱 障礙 3：多條事業線切換太快 — 影像、保險、社群的表達場景完全不同
🏁 中間目標：選定一條線先練熟，再複製到其他線
行動：選擇「影像提案」作為主練習場景（頻率最高、客單價最高、最有 ROI），練30天再擴展到保險和社群
🧱 障礙 4：沒有即時回饋 — 不知道自己有沒有做到
🏁 中間目標：建立「事後復盤3問」的習慣
行動：每次重要對話後花2分鐘回答：①我的 Why 達成了嗎？②對方的反應屬於四種的哪一種？③下次我會調整什麼？

執行順序：障礙3（選定練習場景）→ 障礙2（製作口袋卡）→ 障礙1（設定觸發器）→ 障礙4（建立復盤機制）→ 🎯 目標

🪜 Step 8｜四週突破計畫（轉變樹）

📅 第一週：地基 — 建立觸發機制

  • 行動 1（Day 1-2）：用 ABCDE 重寫你的自我介紹，30秒版本。寫完後對著手機錄音，播放聽一次。
  • 行動 2（Day 3）：製作「口袋檢核卡」（Why / 階段 / Key Message / 資訊觀點感受比例 / CTA），放在手機備忘錄或實體卡片。
  • 行動 3（Day 4-7）：這週的每一次「非即興溝通」（客戶會議、提案、社群貼文）前，花3分鐘看一次口袋卡。不用做到完美，只要「看了」就算完成。
✅ 週末驗證：這週有幾次「溝通前看了口袋卡」？目標 ≥ 3 次。不問品質，只問頻率。

📅 第二週：深化 — 在影像提案中實戰

  • 行動 4（每次提案前）：寫下這次提案的 Why（「我期待這次對話的結果是什麼？」）+ 客戶在決策旅程的哪個階段。
  • 行動 5（每次提案後）：做「事後復盤3問」：①Why 達成了嗎？②對方的反應屬於哪種？③下次調整什麼？用手機備忘錄記錄。
  • 行動 6（週末）：回顧本週所有復盤筆記，找出一個重複出現的模式（例如：你是不是每次都跳過「需求探詢」直接講方案？）。
✅ 週末驗證：累積了幾份復盤筆記？找到了什麼重複模式？

📅 第三週：進階 — 資訊×觀點×感受的調配

  • 行動 7：在本週的一次提案中，刻意加入「感受」成分。例如：「說實話，看到你們品牌的潛力卻還沒有被好好展現，我覺得有點可惜。」觀察對方反應有什麼不同。
  • 行動 8：在社群貼文中嘗試一篇「資訊+觀點+感受」三合一的內容。觀察互動數據和以往的差異。
  • 行動 9：在一次保險溝通中，用「期望值管理」先幫客戶定錨合理預期，再談方案。
✅ 週末驗證：加入感受成分後，對方的反應有變化嗎？用四種反應診斷法分類。

📅 第四週：系統化 — 從行動變成習慣

  • 行動 10：整理前三週的所有復盤筆記，提取你自己的「表達 SOP」— 你的版本的 Why-What-How 流程，不是書的版本，是你用起來最順手的版本。
  • 行動 11：為叡思影像設計一份「消費決策旅程表」，每個階段的溝通內容都寫出來，未來每次提案前只需要對照表格。
  • 行動 12：把 ABCDE 自我介紹在一個真實社交場合使用一次，觀察對方的反應。
✅ 最終驗證：「重要溝通前先做 KISS 檢核」是否已經變成不用提醒的自動行為？如果還沒有，延長第四週再一週。

🔄 Step 10｜持續改善循環（POOGI）

瓶頸1：慣性
（開口前不思考） 打破→ 瓶頸2：技能
（不會判斷客戶階段） 打破→ 瓶頸3：聚焦
（多條線的統一定位） 打破→ 🎯 系統升級

瓶頸遷移預測

當前（第1瓶頸）：慣性 — 開口前沒有 Why-What-How 的自動化流程。用四週突破計畫打破。

打破後（第2瓶頸）：技能 — 你會記得「先想後說」，但判斷客戶在決策旅程的哪個位置需要更多實戰經驗。這時候需要重讀本書的實戰篇，特別是提問模組和需求洞察工具。

再打破後（第3瓶頸）：聚焦 — 表達技能提升後，你會發現「四條事業線各有各的 Key Message」這件事本身就是最大的效率殺手。這時候需要的不是表達技巧的書，而是商業模式設計的書（如《一個人的獲利模式》或與導師的策略對話）。

讀書的目的不是「知道更多」，而是「打破你卡住的那個點」。
這本書幫你打破的是第1瓶頸：從「直覺式表達」升級為「系統化表達」。
打破後，下一個瓶頸會自然浮現 — 那是另一本書的任務。
